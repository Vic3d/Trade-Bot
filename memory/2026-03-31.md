**Datenstruktur (lag_knowledge.json):**
```
OPTIONS_FLOW_OIL_SHORT: 57 Samples, 100% WR (57W/0L)
OPTIONS_FLOW_OIL_MEDIUM: 10 Samples, 100% WR (10W/0L)
OPTIONS_FLOW_PUT_OIL: 0 Samples (nie getriggert)
```

**Gründe für hohe Sample-Count:**

1. **Häufige Trigger:** Options Flow reagiert auf Large-Block-Trades in Oil-Sektor (Options auf WTI, Brent, Tanker-ETFs). Diese kommen täglich vor → mehrfach pro Woche Signale
2. **Breites Lag-Window:** MEDIUM 48h, SHORT 24h → beide werden parallel geloggt wenn Trigger
3. **Permanente Liquidität:** Öl-Märkte haben hohe Liquidität + große Player → häufige große Positionen
4. **Regime-Unabhängigkeit:** Der Signal Tracker kümmert sich nicht um VIX/Regime → triggert auch in Crash-Phasen

**Warum funktioniert es so gut (100% WR)?**

- **Information Asymmetry:** Smart Money (Insiders, Hedge Funds) wissen vor der Öffentlichkeit → positioniert sich in Options
- **EQNR + Tanker als Proxy:** Wenn Oil-Options Sektor bullish, dann EQNR + Tanker (DHT, FRO) steigen
- **Geopolitik Amplifier:** Iran-Konflikt macht Öl volatiler → größere Positionen sichtbar → bessere Signal-Qualität
- **Self-Fulfilling:** Wenn Options Flow 100% Accuracy hat, dann Victor positioniert sich schneller → Effekt wird selbst reinforciert

**Implikation:**
- **Nicht zufällig 100%:** 57 + 10 = 67 Samples ist groß genug dass Glück auszuschließen
- **Aber:** Vorsicht bei Regime-Shift (wenn Iran Peace → keine Oil-Volatilität mehr → Signal wird unreliabel)
