## Lern-Bot — PDF.js Rewrite Phase 1 (09:30–10:30 UTC)

### Kontext: Problem das behoben werden sollte
- Überschriften wurden 4× vorgelesen (AKAD-PDFs haben Überschriften 4× wiederholt im PDF-Encoding)
- `unpdf` concateniert alle Items zu einem String → "1111" und "Statik ebener TragwerkeStatik ebener Tragwerke..." als ein langer String
- Bisherige Fix-Strategie: Regex-basiertes Decoding (`decodeRepeated`, `isRepeatedLine`)
- Viele Fixes (v0.8.0–v0.8.4), aber immer noch fragil

### Entscheidung: Vollständiger Rewrite auf `pdfjs-dist` direkt

**Erkenntnis aus Analyse:**
- `pdfjs-dist` ist bereits installiert (v5.5.207), `unpdf` nutzt es intern nur ohne Positionsdaten
- PDF.js `getTextContent()` liefert pro Item: `transform[4]=x, transform[5]=y, height, str`
- AKAD-Encoding im PDF: "Statik ebener Tragwerke" erscheint als **4 separate Text-Items** an (fast) gleicher y-Position — NICHT als ein langer String!
- `unpdf` concateniert diese Items → "Statik ebener TragwerkeStatik ebener Tragwerke..." (langer String)
- Mit PDF.js direkt: Gruppierung nach y-Position → 4 identische Items = Encoding erkannt, dedupliziert

**Technischer Ansatz:**
- Header-/Footer-Zone: positionsbasiert (y > 92% = Header, y < 8% = Footer) → KEIN Regex mehr
- Encoding-Erkennung: N≥2 identische Items in einer Zeile → `isEncoded=true`, `deduped=items[0].str`
- Kapitel-Erkennung: encoded NUM ("1","1","1","1") gefolgt von encoded Titel ("Statik...") → Heading
- y-Gap-Check: pendingNumber und Titel müssen < 50pt auseinander liegen (verhindert Diagramm-Label-False-Positives wie "2 Teilsystem")

### Worker-Setup für Node.js / Vercel
